## Intro

J'ai toujours eu du mal à mettre en place un environnement de test avec Ansible. En effet, il faut déjà installer Ansible sur sa machine hôte ou sur le serveur qui dirigera les opérations. Ensuite, il faut monter une machine de test qui subira les changements apportés par Ansible. 

Dans la communauté Ansible, il y a un membre très connu pour ses vidéos et ses playbooks du nom de _Jeff Geerling_. Il a même écrit un livre sur le sujet que je vous recommande chaudement et qui est une référence en la matière.

Dans l'épisode 2, Jeff nous montre comment créer une architecture multi VM avec Vagrant. Mais au fait qu'est-ce que Vagrant ?

## Vagrant

Vagrant est un outil édité et maintenu par Hashicorp, un acteur très présent dans le monde devops avec des produits comme: Consul, Vault ou Packer sans oublié Terraform. Ces produits rentrent aisément dans la catégorie Infrastructure As Code ou IAC permettant de créer un infrastructure par le code à l'aide de fichier de configuration.
